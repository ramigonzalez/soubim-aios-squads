**Epic:** 3 — Frontend Dashboard
**Points:** 3
**Priority:** High
**Sprint:** 5

## User Story

**As a** project director reviewing decisions,
**I want** a clear visual distinction between meeting headers and their child decisions,
**So that** I can instantly see the parent-child relationship without needing to collapse/expand.

---

## Acceptance Criteria

### Meeting Card Left Accent Border
- [x] Meeting cards have a `border-l-4` colored by the primary discipline of participants
- [x] Architecture meetings → blue-400 left border
- [x] MEP meetings → orange-400 left border
- [x] Structural meetings → purple-400 left border
- [x] Electrical meetings → amber-400 left border
- [x] Plumbing meetings → cyan-400 left border
- [x] Landscape meetings → green-400 left border
- [x] Orphan group ("Other Decisions") → gray-300 left border

### Decision Rows Inset
- [x] Decision rows area has `bg-gray-50/80` background (contrast with white meeting header)
- [x] Decision rows have slight left indent (`ml-1`) for visual nesting
- [x] Decision rows hover to `bg-white` (lift effect on gray base)
- [x] Decision row background is transparent (inherits gray-50 parent)

### Day Counter Update
- [x] Day header shows "X meeting(s)" instead of "X decisions"
- [x] Pluralization correct: "1 meeting" vs "2 meetings"
- [x] Aria labels updated to reference meetings count
- [x] Works in both date and discipline grouping modes

### New Utility Function

```typescript
export function getDisciplineBorderColor(discipline: string): string {
  const colors: Record<string, string> = {
    architecture: 'border-l-blue-400',
    mep: 'border-l-orange-400',
    structural: 'border-l-purple-400',
    electrical: 'border-l-amber-400',
    plumbing: 'border-l-cyan-400',
    landscape: 'border-l-green-400',
  }
  return colors[discipline.toLowerCase()] || 'border-l-gray-300'
}
```
